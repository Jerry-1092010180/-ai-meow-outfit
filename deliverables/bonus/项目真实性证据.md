# 项目真实性与可验证边界

## 一句话结论

当前项目已经形成可访问、可操作的 H5 商业玩法原型；AIGC 在产品机制中承担角色、动作变体和多人海报的内容生产职责。比赛原型使用确定性 Demo Provider 与效果图保证稳定演示，生产 Provider、API Gateway 和私有 GPU 管线已有清晰接口，但本材料不把“接口存在”描述成“生产模型已经联调完成”。

## 可验证路径

| 验证项 | 操作 | 可观察结果 |
|---|---|---|
| 每日打开钩子 | 打开 `/#/game` | 7 天章节、今日天气/场景、连续收藏卡 |
| 完整穿搭 | 开始今日副本 | 内搭、外套、下装、鞋履、配饰共 5 层 |
| 直觉选择 | 点击商品主卡 | 卡片下沉并显示“已选中”，不跳详情 |
| 商品详情 | 点击“查看详情” | 价格、颜色、尺码、门店等样例字段 |
| AIGC 角色 | 完成或直接预览 | 个人动漫角色、身份分、商品清单 |
| 内容变体 | 切换动作/表情/背景 | 同一身份资产下生成不同内容版本 |
| 好友共创 | 创建房间并复制邀请 | 生成带 `join` 参数的可复用链接 |
| 好友独立身份 | 打开邀请链接 | 好友上传身份、独立选完整 Look、加入同框 |
| 多人场景 | 演示角色加入 | 房间从 1/4 变为 2/4，并解锁互动海报任务 |
| 公开广场 | 首页或结果页 | 用户主动公开；原始身份照片默认不公开 |

## AIGC 的实质作用

AIGC 不是装饰性标签。系统把以下结构化输入组合成内容任务：

```text
AvatarIdentity
+ 5-layer ProductLook
+ DailyEpisodeContext
+ Hair / Pose / Expression / Background
+ SocialRoomMembers
→ Personalized Avatar
→ Content Variations
→ 2–4 Person Social Poster
```

当前演示 Provider 会稳定地产出可展示结果，并保留 `providerStage`；生产接入时由 Gateway Provider 替换，不需要重写页面业务状态。

## Provider 证据

- `src/services/dailyQuestAigcProvider.ts`
  - `DemoPersonalizedQuestProvider`：当前比赛演示主路径。
  - `GatewayDailyQuestAigcProvider`：生产 AIGC 接口。
- `src/services/socialAvatarImageProvider.ts`
  - `effect-preview`：当前角色与多人海报效果演示。
  - `GatewaySocialAvatarImageProvider`：生产图片生成接口。
- `src/services/socialScenePlatformProvider.ts`
  - 本地可复用邀请与场景状态。
  - Gateway 版创建、加入、更新多人场景。
- `server/gateway-worker.js`
  - 浏览器只访问 `/api/avatar/*`；私有 AIGC 不直接暴露公网。
- `server/nerf_head_pipeline_impl.py`
  - 五视角头部实验管线；属于算法储备，不是当前商业 H5 的稳定演示依赖。

## 隐私与安全

- 原始身份照片默认不公开。
- 广场发布需要用户主动开启。
- 对外分享的是动漫角色、昵称和商品清单。
- 生产 AIGC 凭据只进入平台 Secret；浏览器不保存私有 GPU 地址。
