# AI喵搭 — 专项解决方案

> 银泰百货 OPC Bounty 03：为 4500 万会员设计 AI 原生的喵街互动玩法
> 参赛作品：AI喵搭 | 方向：AI穿搭日记 + 3D虚拟试穿 + 社交PK

---

## 一、执行摘要

### 1.1 一句话

**AI喵搭** 是银泰百货喵街 APP 内置的 AI 原生日更互动玩法——AI 根据用户所在地天气、个人身材偏好和门店实时新品，每日生成专属穿搭，用户可 360° 旋转查看 3D 试穿效果，与好友同框 PK，一键分享朋友圈，即看即买。

### 1.2 核心指标目标

| 指标 | 目标 | 计算依据 |
|------|------|---------|
| 日活提升 | +15-20% | 天气驱动的每日穿搭推送钩子 |
| 7日留存 | 40%+ | 连续打卡+穿搭日记资产积累 |
| 分享率 | 25% | 每4次穿搭保存→1次社交分享 |
| 病毒系数(k) | 0.3-0.5 | PK邀请链+朋友圈海报回流 |
| 穿搭→购买转化 | 8-12% | 即看即买，门店库存实时联动 |
| 客单价 | ¥300-500 | 套装推荐，多件组合购买 |

### 1.3 技术亮点

- **AIGC 实质应用**：LLM 风格策划 + Text-to-Image 穿搭生成 + 3D WebGL 实时渲染
- **程序化 3D 人体建模**：根据用户身材参数（5种身型）× 肤色偏好（5种）生成个性化数字人
- **真实天气驱动**：浏览器 GPS → Open-Meteo API → 温湿度感知 → 场景化推荐
- **双人 3D 同框 PK**：WebGL 实时渲染两人穿搭对比，任意角度旋转

## 二、用户研究与业务洞察

### 2.1 目标用户画像

```
姓名：小雯
年龄：26岁
职业：互联网运营
城市：杭州
喵街行为：中低频用户，月均打开 3-5 次
痛点：
  - 每天早上不知道穿什么，翻衣柜 15 分钟
  - 想知道银泰有什么新品，但懒得逛
  - 穿搭分享朋友圈获赞很开心，但缺少"发什么"的灵感
  - 朋友约逛街频率低，一个人懒得去门店
```

**核心发现**：18-40 岁女性用户的穿搭决策是**高频刚需**（每天都要穿衣服），传统方式缺少**惊喜感**和**社交性**。

### 2.2 喵街 APP 业务痛点

| 痛点 | 现状 | AI喵搭解法 |
|------|------|------------|
| 低频打开 | 会员月均打开 3-5 次 | 每日天气穿搭推送，制造"今天穿什么"的期待感 |
| 缺乏互动 | APP 以交易为主，用完即走 | 穿搭日记 + PK挑战 + 社交分享，停留时长 3-5 分钟 |
| 线上线下割裂 | 线上浏览和线下试穿分离 | AI推荐穿搭→门店库存联动→到店试穿导航 |
| 分享动力弱 | 优惠券分享打开率低 | AI生成时尚海报+PK战报，天然适合社交媒体传播 |

### 2.3 竞品分析

| 产品 | 优势 | 劣势 vs AI喵搭 |
|------|------|---------------|
| 小红书 | 海量穿搭UGC | 无门店库存联动，不能即看即买 |
| 蘑菇街 | 穿搭社区成熟 | 无 AI 个性化生成，无 3D 试穿 |
| 得物 | 潮流社区活跃 | 偏男性球鞋，无百货全品类 |
| Whering(海外) | AI衣橱管理 | 无门店联动，无社交PK |
| **AI喵搭** | **天气×AI×门店×3D×社交 五位一体** | 需依托喵街 APP 生态 |

## 三、产品设计

### 3.1 核心游戏循环（Hook Model）

```
触发 (Trigger)
  ├─ 外部：推送通知 "今日杭州 32°C 晴天，AI为你推荐穿搭"
  └─ 内部：用户惦记"今天的穿搭是什么？"

行动 (Action)
  └─ 打开 APP → 选择心情 → AI 生成穿搭 (3秒加载)

多变的奖励 (Variable Reward)
  ├─ 穿搭可能惊艳/有趣/惊喜（每天不同）
  ├─ 风格评分高低不一，驱动"再试一次"
  └─ 随机掉落优惠券

投入 (Investment)
  ├─ 保存到穿搭日记（积累个人资产）
  ├─ 连续打卡（损失厌恶）
  ├─ 完善风格档案（推荐越来越准）
  └─ 发起 PK（社交投入）
```

### 3.2 功能模块

#### 模块A：每日穿搭生成（核心钩子）

```
用户打开 APP
  → 实时天气展示（城市/温度/湿度/天气状况）
  → 选择今日心情（6种: 开心/平静/活力/随性/浪漫/自信）
  → AI 生成动画（4步: 分析偏好→匹配天气→精选新品→穿搭就绪）
  → 穿搭卡片揭秘（翻转动画）
  → 操作: 保存日记 / 换一套 / 分享 / 买这套
```

**AIGC 角色**：LLM 根据天气+心情+用户风格策划搭配逻辑；Text-to-Image 生成穿搭效果图。

#### 模块B：3D 虚拟试穿

```
拍照/跳过 → AI 处理(4步动画) → 3D 人体模型展示
  → 360° 拖拽旋转
  → 滚轮/捏合缩放
  → 5 套穿搭切换
  → 分享
```

**技术实现**：Three.js WebGL 程序化人体（17个几何体拼合），根据用户真实身材参数（bodyType）变形，肤色（colorPreference）映射。

#### 模块C：穿搭日记

```
日历视图 → 每月穿搭热力图
时间线 → 历史穿搭卡片
统计 → 总搭配数/连续打卡/平均评分
```

#### 模块D：PK 挑战 + 3D 同框

```
发起挑战
  → 选择主题（10个）
  → 生成 6 位邀请码
  → 分享微信/朋友圈
  → 好友点击链接加入
  → 两人 3D 模型同框展示
  → 社区投票（24h）
  → 胜者获得优惠券
```

#### 模块E：门店好物（转化）

```
AI推荐穿搭 → 每件单品链接 → 实时库存 → 门店导航 → 一键购买
```

### 3.3 用户旅程图

```
Day 1: 下载喵街 APP → 3 步引导（身型/风格/门店）→ 首套 AI 穿搭 → 保存日记
Day 2: 推送 "杭州 28°C 多云" → 打开 → 心情 → 新穿搭 → 分享朋友圈
Day 3: 好友通过分享链接进来 → 发起 PK → 两人 3D 同框 → 朋友圈投票
Day 7: 连续 7 天打卡 → 获得徽章 → 风格报告 → 推荐门店新品 → 下单
Day 30: 30 天穿搭日记 → AI 月报 → 风格演变 → 高黏性用户
```

## 四、AIGC 技术架构

### 4.1 AI 管线设计

```
输入层
  ├── 用户风格画像 (bodyType, styleTags, colorPreferences)
  ├── 实时天气 (temperature, humidity, condition, season)
  ├── 用户心情 (6 种 mood)
  └── 门店新品库存 (50+ SKU 实时)

        ↓

LLM 风格策划层 (Claude API / 通义千问)
  ├── Prompt: "你是银泰百货时尚顾问。根据 [天气] [心情] [用户风格]
  │    从 [门店库存] 中策划一套完整穿搭。"
  └── Output: JSON { outfitName, pieces[], styleDescription, score }

        ↓

Text-to-Image 生成层 (Stable Diffusion / Midjourney API)
  ├── Prompt: "Fashion editorial photo, [bodyType] model wearing
  │    [pieces], [styleDescription], studio lighting, 8K"
  └── Output: 穿搭效果图 URL

        ↓

3D 渲染层 (Three.js WebGL)
  ├── 程序化人体建模 (17 个几何体, 根据 bodyType 变形)
  ├── 材质着色 (topColor, bottomColor, shoeColor)
  └── 实时渲染 (60fps, OrbitControls, shadows)
```

### 4.2 原型阶段 Mock 策略

- **LLM**：预定义 15 套穿搭 JSON（覆盖 6 种心情 × 4 季天气），模板化风格描述
- **Text-to-Image**：30 张预生成穿搭图（picsum placeholder），服务接口与真实 API 一致
- **3D 模型**：纯程序化生成，零外部依赖，后续可升级加载真实 GLB 模型

**迁移到生产时**，只需替换 `outfitService.ts` 中的实现，前端代码无需改动。

### 4.3 AI 实质性判定（评审关键）

| 评审维度 | 证据 |
|---------|------|
| **AI 是否为核心** | 拿掉 AI，产品不存在——穿搭策划、图像生成、风格分析均依赖 AI |
| **AI 贡献占比** | 70%+ 的用户体验由 AI 驱动（生成→渲染→推荐） |
| **AI 不可替代性** | 人工不可能为 4500 万用户每人每日生成个性化穿搭 |
| **AI 输出质量** | 多模态输出（文本+图像+3D），场景化、个性化、实时化 |

## 五、前端技术架构

### 5.1 技术选型

| 层次 | 技术 | 选型理由 |
|------|------|---------|
| 框架 | React 18 + TypeScript | 生态成熟，H5 首选，评委可直接打开 |
| 构建 | Vite 5 | 秒级热更新，生产构建 <500ms |
| UI | TailwindCSS 4 | 原子化 CSS，移动端适配便捷 |
| 动画 | Framer Motion 11 | 卡片翻转、页面过渡、手势动画 |
| 状态 | Zustand + persist | 轻量，localStorage 持久化，原型零后端 |
| 3D | Three.js + React Three Fiber | WebGL 工业标准，2M+ GitHub Stars |
| 路由 | React Router v6 | 懒加载代码分割 |
| 天气 | Geolocation API + Open-Meteo | 免费、无需 API Key、全球覆盖 |
| 分享 | Web Share API + Canvas 海报合成 | 客户端生成，零服务器成本 |

## 六、社交分享设计

### 6.1 分享链路

```
用户保存穿搭 → 生成 AI 时尚海报 → 选择平台
  ├── 微信好友 (Web Share API / JS-SDK)
  ├── 朋友圈 (Canvas 合成图片 + 长按保存)
  ├── 小红书 (deeplink 跳转)
  └── 复制链接 (miaojie:// scheme + H5 兜底)

好友点击链接
  ├── 已安装喵街 → deeplink 打开 APP → 进入挑战/穿搭页
  └── 未安装 → H5 落地页 → "下载喵街APP体验AI试穿"
```

### 6.2 裂变机制

- **PK 邀请链**：每次 PK ≥ 2 人参与 + N 人投票 = 病毒传播
- **穿搭海报**：AI 生成精美时尚卡片，天然适合朋友圈传播
- **月报分享**："我的 7 月穿搭报告" → 身份认同 → 主动传播

## 七、门店融合与商业闭环

### 7.1 O2O 转化漏斗

```
线上发现 (AI推荐穿搭)
  → 商品详情 (品牌/价格/库存/门店)
  → 加入购物车 (多个单品一键加购)
  → 下单购买 (喵街现有交易流程)
  → 到店自提/配送
  → 穿搭返图 (UGC 回流)
```

### 7.2 门店引流

- **穿搭含门店信息**：每件单品标注"杭州武林银泰 2F 有售"
- **到店试穿导航**：一键跳转门店地图
- **门店专属挑战**："武林银泰限定穿搭PK"→ 到店打卡
- **优惠券核销**：PK 获胜得门店优惠券 → 到店消费

### 7.3 商业价值测算

```
假设:
  - 喵街 MAU 500 万
  - AI喵搭渗透率 20% = 100 万用户
  - 穿搭→购买转化率 10%
  - 客单价 ¥400
  - 月均购买频次 1.5 次

月 GMV = 100万 × 10% × ¥400 × 1.5 = ¥6000 万/月
年 GMV = ¥7.2 亿
```
