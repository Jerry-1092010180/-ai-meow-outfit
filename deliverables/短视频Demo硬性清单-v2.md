# 短视频 Demo 硬性清单 v2

> 这是项目内部的交付硬标准；若赛事官方格式另有要求，以官方规则为准。

## 成片规格

- 建议时长：120 秒，绝不超过 3 分钟。
- 分辨率：1920×1080，30 fps，H.264 MP4。
- 手机画面使用真实 H5 操作录屏，不用静态 PPT 冒充运行效果。
- 字幕必须可独立理解，静音观看仍能看懂完整闭环。
- 所有个人穿搭生成占位图明确标记“AIGC 效果示意”。

## 120 秒镜头表

| 时间 | 画面 | 必须说清 |
|---|---|---|
| 0–10s | 中低频会员打开喵街却没有每日理由 | 一句话问题：缺少每日内容和分享动机 |
| 10–22s | 今日副本大厅、七日剧情、次日预告 | 每天不是重复签到，而是新角色、新场景、新商品池 |
| 22–38s | 60 秒三轮选择连续点击 | 用户只需三次直觉决策，门槛低 |
| 38–50s | AI 生成过程和输入轨迹 | 天气、风格、库存、今日场景进入生成 |
| 50–70s | A/B 结果切换和个人效果示意 | A 保留三件；B 保留一件、AI 替换两件 |
| 70–84s | 发起者分享链接 | 问题是“哪套更像我”，不是机械助力 |
| 84–98s | 第二台手机/独立窗口完成好友投票 | 好友看到同一组 A/B，并选择最终封面 |
| 98–110s | 发起者奖励解锁、同款和楼层信息 | 社交动作连接到店券与银泰真实商品 |
| 110–120s | 漏斗和一句收尾 | 每日打开 → AI 内容 → 社交传播 → 到店复购 |

## A/B 镜头硬要求

- 必须先拍到结果页对 A/B 来源的文字解释。
- 必须点击 LOOK A 和 LOOK B，证明不是一张图换标题。
- 画面中至少有两件商品发生变化。
- 好友端必须再次显示：A=她的选择，B=AI 换两件。
- 不使用“裁决”一词而不解释裁决对象。

## AIGC 实质应用镜头硬要求

- 出现输入：天气、用户风格、门店库存、用户三次选择。
- 出现推理过程：识别身份单品、生成保留版、生成反转版。
- 出现结构化输出：两套商品、分数、理由、效果图和可购同款。
- 生产 Provider 尚未接通前，视频中不得伪造真实模型日志或 GPU 请求。
- 最终接通后补拍真实 `traceId`、模型版本、生成耗时和 CDN URL。

## UI 图必须覆盖

1. 今日副本大厅：七日连续剧、天气、门店、奖励、次日预告。
2. 三轮商品选择：商品、品牌、价格、门店楼层和倒计时。
3. AI 生成：四步处理轨迹。
4. A/B 结果：两套来源说明、可切换海报和评分。
5. 好友投票：两套完整造型和投票按钮。
6. 奖励与转化：好友完成、到店券、同款和门店任务码。

## 禁止出现

- 旧版木乃伊、圆柱人体、反向脸贴图或破碎 3D 模型。
- 把确定性 Demo Provider 描述成已经在线运行的大模型。
- 只讲技术架构，不展示每日钩子和好友实际动作。
- 未解释来源的 A/B。
- 未经实验验证的“DAU 必然增长 20%”等承诺。
- 纯概念视频、无法点击的静态页面或虚构核销结果。

## 录制前验收

- 从无缓存浏览器完整走一遍发起者流程。
- 用第二浏览器打开复制出的分享链接并投票。
- 确认 A/B 商品 ID 与发起者结果一致。
- 确认移动端底部按钮不被浏览器工具栏遮挡。
- 确认所有品牌、价格和门店文字可读。
- 确认“效果示意”和“真实已实现”边界在字幕中出现。
- 保留一份无旁白纯操作录屏，方便后期重新剪辑。
