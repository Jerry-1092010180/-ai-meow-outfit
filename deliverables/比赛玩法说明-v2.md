# AI 喵搭：银泰每日角色副本

## 核心提案

把“今天穿什么”变成喵街每天更新的一集角色故事。用户用照片建立可持续复用的动漫身份，再从内搭、外套、下装、鞋履和配饰中完成整套选择，AIGC 生成“像自己”的今日角色。用户可以继续换发型、动作、表情和背景，再邀请好友带着自己的角色进入 2–4 人同框。

## 为什么用户愿意每天打开

1. **每天剧情不同**：日期、天气、商圈和门店事件共同生成今日章节。
2. **角色资产持续成长**：身份不重建，今天只更新穿搭与内容版本。
3. **7 天连续收藏**：完成一集获得数字衣橱卡；连续章节解锁新场景。
4. **商品池每日变化**：新商品让同一角色产生新的视觉结果。
5. **好友房间有后续**：共创房间可以继续添加成员、动作和场景，不是一次性分享页。

“约 60 秒完成”是降低参与门槛的承诺，不是限时规则；页面没有倒计时。

## 用户主流程

1. 打开今日章节，看到剧情、天气和商圈。
2. 上传正面照，建立本次动漫身份。
3. 依次选择内搭、外套、下装、鞋履和配饰。
4. 点击商品卡只做选中；点击“查看详情”才进入价格、尺码和门店字段。
5. AIGC 生成个人角色、动作和分享海报。
6. 用户编辑发型、姿势、表情和背景。
7. 用户分享房间链接，好友创建自己的角色与完整 Look。
8. 2–4 人选择并肩逛街、碰拳击掌、一起走秀或合照自拍。
9. 用户可主动公开到银泰穿搭广场；其他用户可发现同款并回商品详情。

## 社交机制

好友不是评委，也不是助力按钮。

```text
发起者角色 + 发起者 Look
→ 生成房间邀请
→ 好友上传自己的身份
→ 好友选择自己的完整 Look
→ 好友角色加入 2–4 人场景
→ 生成新的互动海报
→ 海报继续带来下一位参与者
```

每次分享都能新增一个角色、一套商品选择和一张多人内容，因此分享动作本身会扩展内容，而不是只增加一次点击。

## AIGC 的不可替代作用

- 将用户身份统一成可识别的漫画角色视觉。
- 将 5 层商品与场景组合成完整人物内容。
- 根据动作、表情和背景生成不同内容版本。
- 保持 2–4 位成员各自身份和商品，完成互动构图。
- 为公开广场生成标题、标签和商品映射。

如果移除 AIGC，产品只剩普通商品清单，无法形成每日内容、个人表达与多人共创。

## 经营闭环

```text
每日打开
→ 选择商品
→ 生成角色
→ 分享邀请
→ 好友新建角色
→ 多人海报 / 公开广场
→ 看同款与商品详情
→ 到店任务
→ 次日新章节
```

建议核心指标：章节打开率、角色生成完成率、邀请发送率、好友角色完成率、多人海报生成率、同款详情点击率、门店任务领取率和次日回访率。

## 当前原型边界

- 已运行：每日章节、5 层选择、选中下沉、独立详情、角色编辑、邀请链接、2–4 人席位、公开广场。
- 效果演示：当前角色图、动作变体和多人海报。
- 生产接入：银泰 PIM/库存、Gateway AIGC、账号关系、CDN、审核与正式奖励发放。

## 升级路线：动态商品展示视频

从静态海报扩展为 5–15 秒动态数字人商品展示视频：

- **风格支持**：写实真人商品展示 / 漫感 2.5D 电子角色展示。
- **动作支持**：站姿展示、走秀、转身、打招呼、指向商品。
- **多人同框**：2–4 人各自保持角色与穿搭，可互动同框。
- **可实施路径**：Identity Lock → Garment Lock → Motion Reference → Validation Gate，详见 [`动态试穿视频技术路径.md`](./动态试穿视频技术路径.md)。
- **当前状态**：技术路线已文档化，待 Q3 生产联调接入。
